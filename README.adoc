= Evolution Engine
:revdate: 2026-03-21
:toc: left

The core transformation engine for the **Evolution Adaptation Ecosystem**. A high-performance, D-based platform for automated, rule-driven codebase evolution.

== Features

* **Generic & Modular**: Decoupled from specific rulesets. Use it for Qt, OpenCV, React, or any framework.
* **Version Traversal**: Automatically finds transformation paths between versions (e.g., `--from 1.0 --to 2.0`).
* **SDL Rules**: Uses human-readable SDL (Simple Declarative Language) for defining transformation logic.
* **Zero Dependencies**: Compiled D binary for near-instant execution in any environment.

== Installation

Requires the `dub` package manager (part of the D compiler suite).

[source,bash]
----
git clone https://github.com/AMDphreak/evolution-engine.git
cd evolution-engine
dub build -b release
----

== Usage

[source,bash]
----
./evolution-engine --path "/path/to/code" \
                   --rules-dir "/path/to/rules" \
                   --from "1.0" \
                   --to "2.0"
----

=== CLI Options

* `-p, --path`: Path to the source code to adapt (folder or file).
* `-r, --rules-dir`: Directory containing version-specific rules (`[from]-[to].sdl`).
* `-f, --from`: The starting version of your codebase.
* `-t, --to`: The target version to evolve to.
* `-e, --extensions`: Comma-separated list of file extensions (default: `.py,.cpp,.h`).
* `-d, --dry-run`: Preview changes without modifying files.

== GitHub Action

For seamless CI/CD integration, use the modular **link:https://github.com/AMDphreak/evolution-action[Evolution Action]**. It automatically fetches the engine and your specified ruleset to adapt code during your build pipeline.
